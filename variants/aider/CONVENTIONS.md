# Aider Conventions

- Read `CODER.md`, `PLAN.md`, `TODO.md`, and `CHANGELOG.md` before non-trivial work.
- Audit before implementation.
- Plan non-trivial work in `PLAN.md`.
- Prefer the smallest effective change over broad rewrites.
- Preserve existing project patterns and naming.
- Read `FRONTEND-DNA.md` for frontend work and `BACKEND-SECURITY-CHECKLIST.md` for sensitive backend work.
- Verify before claiming completion.
- Be explicit about assumptions, trade-offs, and residual risk.
