# Cline Core Rules

- Read `CODER.md`, `PLAN.md`, `TODO.md`, and `CHANGELOG.md` before non-trivial work.
- Audit before implementation.
- Use `PLAN.md` for non-trivial execution plans.
- Prefer minimal diffs and preserve existing project structure.
- Read `FRONTEND-DNA.md` before meaningful UI changes.
- Read `BACKEND-SECURITY-CHECKLIST.md` before sensitive backend changes.
- Verify before claiming completion and report residual risk honestly.
