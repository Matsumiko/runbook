# Progression And Step States

Build onboarding to create momentum without confusion.

Prefer patterns like:

1. clear welcome context
2. visible next action
3. short, focused steps
4. explicit progress or completion signal
5. immediate transition to the next meaningful state

Keep rules like these in mind:

- one step should usually answer one user question or request one real action
- do not hide whether a step is optional, skippable, blocked, or complete
- keep primary CTAs direct and action-oriented
- back, skip, and resume behavior should be understandable, not surprising
- if the flow is long, use progress signals that reduce anxiety instead of adding ceremony
- on mobile, keep the current step title, progress cue, and primary CTA visible before secondary explanation
