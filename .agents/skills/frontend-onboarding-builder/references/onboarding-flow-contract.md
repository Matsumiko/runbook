# Onboarding Flow Contract

Define the onboarding flow contract before composing steps.

Clarify:

- who is entering the flow and what account or data state they are in
- what "activated" means for this product
- which setup steps are required versus optional
- what the user should understand in the first viewport
- what action creates first real value
- whether the flow is step-by-step, checklist-based, or section-based
- what happens when the user skips, pauses, or returns later

Prefer the shortest credible path to first value. Do not expand onboarding just because more preferences exist in the product.
