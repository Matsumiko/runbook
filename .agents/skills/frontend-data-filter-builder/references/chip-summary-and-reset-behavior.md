# Chip Summary And Reset Behavior

Applied filters should be easy to understand and undo.

## Chip summary

- Show active constraints in the same language users chose them.
- Avoid cryptic abbreviations unless the surrounding product already uses them.
- Keep chip density subordinate to the result surface.

## Reset behavior

- Clear-one should not feel hidden.
- Clear-all should be easy to find when multiple constraints are active.
- Distinguish returning to default state from removing every possible filter forever.

## Empty results

- Over-filtered results should explain how to recover.
- Do not confuse no data with too many active constraints.
