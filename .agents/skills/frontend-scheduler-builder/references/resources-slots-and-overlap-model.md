# Resources, Slots, and Overlap Model

Schedulers live or die on density clarity.

Guidelines:

- make the resource axis and the time axis independently scannable
- keep slot height or width consistent enough to compare occupancy
- distinguish overlap, blocked time, and overflow instead of letting them blur together
- use badges, avatars, and status colors only when they clarify assignment meaning
- keep unassigned or unscheduled work separate from confirmed bookings

If overlap becomes unreadable, reduce card detail before weakening time or resource meaning.
