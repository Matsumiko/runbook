# Pane Layout Contract

Define the split-pane shell before placing content.

Capture:

- pane count and responsibility
- primary versus secondary pane
- default widths and min-width rules
- resize, collapse, pin, or swap behavior
- scroll ownership per pane
- pane headers, toolbars, or tabs
- state coverage: loading pane, empty pane, unavailable pane, and collapsed pane

Make the contract explicit about whether the panes are persistent, temporary, or responsive-only.
