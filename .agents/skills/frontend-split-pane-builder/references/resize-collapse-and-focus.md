# Resize, Collapse, and Focus

Split panes need trustworthy interaction rules.

Guidelines:

- resize affordances should be discoverable without dominating the layout
- collapsed panes should have a clear restore path
- keyboard focus should remain predictable after pane changes
- scroll containers and focus rings should not fight each other
- pane headers and actions should remain associated with the correct pane after resize

If resize is not implemented, do not imply draggable behavior.
