# Explanation And Next Actions

Build empty states to answer three questions quickly:

1. what happened
2. why it matters
3. what to do next

Prefer a structure like:

1. concise title
2. short explanation grounded in the actual state
3. one primary next action
4. one or two secondary actions only if they are genuinely useful

Keep rules like these in mind:

- lead with the most actionable explanation, not a vague placeholder
- keep the primary CTA tied to recovery, creation, reset, import, invite, or support
- do not stack multiple equally strong actions
- keep helper copy honest and calm rather than overly cheerful
- if the user cannot act, explain the limit clearly instead of pretending an action exists
