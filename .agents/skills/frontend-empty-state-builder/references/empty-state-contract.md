# Empty State Contract

Define the empty-state contract before composing the surface.

Clarify:

- what the user expected to find
- why that content or capability is missing right now
- whether the state is no data, no results, blocked access, unavailable capability, or not-yet-configured setup
- what action would create progress from here
- whether recovery is immediate, deferred, or impossible for this user
- whether the state lives as a full page, section card, table replacement, or search result shell

Prefer the smallest empty-state surface that restores user momentum. Do not inflate it with decorative content that distracts from the next action.
