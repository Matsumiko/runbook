# Auth Flow Contract

Define the auth flow contract before building the surface.

Clarify:

- which step the user is in
- which credentials, codes, or tokens are required
- where success should send the user
- which failures need explicit recovery
- whether messaging should stay privacy-safe
- what support, resend, or retry actions must exist

Prefer the smallest auth surface that completes the current identity task clearly. Do not mix unrelated product onboarding or marketing content into the same flow unless the product already does that.
