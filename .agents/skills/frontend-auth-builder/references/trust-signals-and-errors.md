# Trust Signals And Errors

Auth UI must feel trustworthy before it feels polished.

Keep rules like these in mind:

- use calm and direct copy
- keep brand and product cues visible enough that the user knows where they are
- prefer privacy-safe errors where revealing account status would be risky
- keep recovery links available but secondary
- make resend, retry, and support paths obvious when the user is blocked

Avoid:

- playful or vague microcopy during auth failure
- over-sharing whether an account exists when the backend contract expects privacy-safe behavior
- dense decorative layouts that weaken trust in the primary action
