# Field States And Selection Model

Select controls need field clarity first.

## Required field states

- Empty with placeholder
- Focused and open
- Selected and closed
- Disabled
- Error or validation feedback

## Selection clarity

- Keep placeholder and committed value visually distinct.
- Multi-select should keep chip or count treatment stable after selection.
- If grouped options exist, make the grouping legible without overpowering the field label.

## Validation

- Error messaging should sit where nearby fields place validation feedback.
- Do not hide invalid or required-state meaning inside the open menu only.
