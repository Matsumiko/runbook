# Priority, SLA, and Triage Actions

Queue work is about fast decisions.

Guidelines:

- keep urgency and SLA cues immediately scannable
- separate severity from unread or recently updated state
- use fast actions only for the workflows that truly need them
- avoid forcing users to open detail views just to classify obvious items
- keep ownership and escalation paths visible without turning every row into a control bar

If a priority cue is domain-specific, label it clearly instead of relying on color alone.
