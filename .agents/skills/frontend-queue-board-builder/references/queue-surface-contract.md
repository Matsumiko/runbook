# Queue Surface Contract

Define the operational queue before choosing list or board treatment.

Capture:

- work-item type
- ordering or grouping rules
- urgency, priority, owner, and due-state semantics
- selection and bulk-action model
- quick triage actions
- state coverage: loading, empty queue, filtered-empty, blocked, escalated, and error

The contract should explain why this is a queue and not merely a table of records.
