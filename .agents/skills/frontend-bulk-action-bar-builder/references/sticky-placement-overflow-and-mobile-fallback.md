# Sticky Placement, Overflow, And Mobile Fallback

- Choose sticky placement based on where users already look for mass actions.
- Preserve the readability of the underlying table, queue, or list when the bar appears.
- Move secondary actions into overflow only after the core selection summary stays readable.
- On smaller screens, collapse the action set into sheets or compact menus while keeping selected count visible.
- Distinguish selection lost due to filters or pagination from explicit clear-selection actions.
