# Page Contract

Define the page contract before composing sections.

Clarify:

- who arrives on the page
- why they are here
- what they should understand in the first viewport
- what primary action or primary read should happen first
- which sections are essential versus optional
- what page states matter: loading, empty, error, gated, or unauthenticated

Prefer the smallest page that completes the user's main job clearly. Add supporting sections only if they reinforce that job.
