# Page States And Entry Points

Treat page states as part of the product flow.

When building a page, account for:

- first-load behavior
- no-data or not-yet-configured states
- error or unavailable states
- gated, unauthenticated, or permission-limited entry
- deep-link entry where the user skips previous context

Entry-point checklist:

- can the user understand where they are immediately
- is the first available action clear
- does the page still make sense when some supporting data is missing
- does the page remain usable when sections collapse on narrow screens
