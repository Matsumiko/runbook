# Keyboard Touch And Discoverability

Right-click cannot be the only access path for important actions.

## Keyboard fallback

- Provide a keyboard invocation path when the product already supports keyboard-heavy workflows.
- Keep focus return and close behavior predictable after dismissal.

## Touch fallback

- Long press is acceptable only if the product supports it coherently.
- If long press is obscure, provide an alternate visible trigger on touch devices.

## Discoverability

- Context actions should not feel like hidden features unless that is intentional and acceptable in the product.
- In dense admin surfaces, subtle visible affordances can coexist with context-click support.
