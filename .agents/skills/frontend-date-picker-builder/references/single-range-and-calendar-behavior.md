# Single Range And Calendar Behavior

Date-selection clarity matters more than calendar decoration.

## Single-date behavior

- Make the current committed date obvious.
- Keep invalid typed input visually distinct from empty state.
- Prevent impossible selections when bounds are known.

## Range behavior

- Make start, hover-preview, and completed range states easy to distinguish.
- Clarify whether the user can edit only the start, only the end, or either endpoint.
- If presets exist, make it obvious when preset selection overrides custom range.

## Calendar navigation

- Keep month or period navigation predictable.
- Avoid dense chrome that competes with the calendar grid.
- Use dual-month layouts only when the task genuinely benefits from broader range visibility.
