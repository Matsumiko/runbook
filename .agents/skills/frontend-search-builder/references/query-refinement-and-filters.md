# Query Refinement And Filters

Build search controls to make narrowing results feel clear, not heavy.

Prefer a structure like:

1. primary query input
2. visible result summary or active scope
3. the few highest-value refinements
4. optional advanced filters behind a disciplined drawer or panel
5. reset or clear path when constraints pile up

Keep rules like these in mind:

- make the active query and active filters easy to scan
- keep filter labels task-oriented, not backend-oriented
- do not expose every possible refinement in the first row
- use chips, badges, or summaries to reflect active constraints honestly
- if the surface has saved or recent searches, keep them supportive rather than dominant
- on mobile, preserve the query, active filter count, and access to refinement controls before dense result content
