# Overlay Focus and Empty States

Command palettes often fail on lifecycle details rather than visuals.

Verify:

- focus lands in the query field on open when appropriate
- keyboard movement, selection, and close behavior are explicit
- empty or no-match states offer recovery instead of a dead end
- unavailable or permission-limited commands are explained honestly
- execution feedback is clear when selection triggers an action, not just navigation

Treat the open-search-close loop as one continuous interaction.
