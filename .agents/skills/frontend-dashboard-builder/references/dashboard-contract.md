# Dashboard Contract

Define the dashboard contract before building panels.

Clarify:

- who the dashboard serves
- which decisions it should help the user make
- the primary KPI list
- which sections are summary, analysis, or operational follow-up
- which panels need live data versus placeholder structure
- whether the page needs filters, date ranges, saved views, or segment toggles

Prefer the smallest dashboard that answers the main operational questions first. Add secondary panels only if they support the primary workflow.
