# Layout And Hierarchy

Build dashboards around a clear scan path.

Prefer a hierarchy like:

1. page title and control bar
2. top-line KPIs
3. primary chart or primary operational surface
4. supporting sections such as feeds, alerts, tables, or breakdowns

Keep rules like these in mind:

- do not make every card visually loud
- keep related panels aligned to a common grid
- use consistent card padding and section spacing
- reserve larger spans for the panels that explain change, not just report totals
- let secondary information collapse or stack sooner on narrow screens
