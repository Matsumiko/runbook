# Responsive and Actions

Data surfaces usually degrade poorly if responsiveness is not designed deliberately.

## Responsive Pass

Check:

- whether the table remains usable on narrower widths
- whether horizontal scroll is acceptable for this product
- whether key columns should collapse into stacked or summary layouts
- whether row actions remain reachable on mobile

Do not assume desktop tables simply scale down.

## Sorting and Filtering

- Make the active sort understandable.
- Keep filters grouped and readable.
- Avoid toolbar clutter.
- Reset and clear actions matter when filters are involved.

## Row and Bulk Actions

- Use row actions when actions are contextual to one record.
- Use bulk actions when users reasonably act on many rows.
- Keep destructive actions clearly separated.
- Avoid menus so hidden that users cannot find important actions.

## Verification Standard

If loading, empty, and responsive behavior were not checked, the table pass is only partial.
