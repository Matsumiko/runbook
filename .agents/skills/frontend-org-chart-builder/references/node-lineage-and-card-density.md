# Node, Lineage, and Card Density

Prioritize hierarchy readability over decorative detail.

Guidelines:

- keep primary identity fields visible without opening a side panel
- show only the metadata needed to compare siblings quickly
- use connector lines, grouping labels, or lane headers only when they clarify parent-child meaning
- avoid oversized cards that break scanability across a large tree
- keep sibling spacing, connector weight, and collapse indicators consistent
- make dotted-line or secondary relationships visibly different from direct reporting lines

If many descendants are visible, reduce metadata before reducing lineage clarity.
