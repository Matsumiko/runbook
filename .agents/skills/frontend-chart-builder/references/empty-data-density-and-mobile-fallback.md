# Empty Data Density And Mobile Fallback

Charts fail quickly under bad data states or compressed layouts.

## Empty or partial data

- Distinguish no data, zero values, and unavailable data.
- Provide concise explanation when the chart cannot render meaningfully.

## Dense data

- Simplify labels, series, or annotations before the chart becomes illegible.
- Prefer summary text or alternate list views when the chart no longer explains itself.

## Mobile fallback

- Preserve the main analytical question when the chart card gets narrow.
- Use simplified labels, reduced legends, or alternate summaries on smaller screens.
