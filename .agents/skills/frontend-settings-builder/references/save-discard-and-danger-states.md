# Save Discard And Danger States

Treat persistence and destructive behavior as core UX.

Account for states like:

- unsaved changes
- save in progress
- save succeeded
- save failed
- permission-limited setting
- dangerous action confirmation
- irreversible account or team changes

Checklist:

- users can tell whether their changes are saved
- destructive actions are clearly separated and confirmed
- permission limits are explained rather than silently blocked
- success and failure states are explicit and recoverable
