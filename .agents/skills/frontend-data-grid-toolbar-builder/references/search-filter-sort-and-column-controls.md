# Search, Filter, Sort, and Column Controls

Toolbar controls should communicate priority.

Guidelines:

- keep the primary query or search input dominant when discovery matters
- keep filter chips and reset affordances close enough to understand current constraints
- use sort and view controls as supporting actions, not visual anchors
- keep column visibility and export secondary unless the product is grid-heavy
- avoid stacking too many equally styled controls across one row

If the toolbar becomes denser than the data surface, regroup or collapse controls before adding more chrome.
