# Checkout Flow Contract

Define the checkout flow contract before building the surface.

Clarify:

- what is being purchased
- which steps are required
- what address, shipping, tax, or billing data is needed
- how totals are calculated and displayed
- which payment methods are available
- where success should send the user
- which failure or retry paths must exist

Prefer the smallest checkout that completes the purchase clearly and safely. Keep optional upsells or extras secondary.
