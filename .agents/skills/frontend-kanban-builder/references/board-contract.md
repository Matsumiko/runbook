# Board Contract

Use this reference to define the workflow surface before implementing lanes.

## Common kanban jobs

- Ticket or issue board
- Sprint or backlog board
- Editorial or content pipeline
- Multi-stage operational workflow

## Contract questions

Clarify:

- what each lane means
- how cards move between lanes
- whether swimlanes or grouping exist
- whether counts or WIP hints matter
- what empty lanes should communicate

## Pattern boundary

- Use table-builder when rows and columns matter more than lane flow.
- Use gantt-builder when start/end span planning matters more than status columns.
