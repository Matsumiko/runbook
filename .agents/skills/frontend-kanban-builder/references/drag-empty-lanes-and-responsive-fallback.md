# Drag Empty Lanes And Responsive Fallback

Board affordances should match real behavior.

## Drag honesty

- If drag or reorder is not implemented, do not style every card as if it is draggable.
- Keep drag handles or movement affordances subtle but discoverable when they exist.

## Empty lanes

- Empty lanes should explain that the stage exists and is currently unused.
- Avoid making empty columns feel broken or collapsed away without intention.

## Responsive fallback

- Many-lane boards may need horizontal scroll, stacked sections, or alternate list fallback.
- Preserve workflow order even when the board collapses on smaller screens.
