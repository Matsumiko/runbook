# Sheets Drawers And Destructive Flows

Choose the overlay pattern that fits the task:

- centered dialog: small to medium focused task or confirmation
- side sheet or drawer: contextual edit or preview that benefits from keeping page awareness
- bottom sheet: lighter mobile action group or compact review task
- destructive confirm: short message, explicit consequence, singular primary action

Guidelines:

- if the task is dense or multi-step, prefer a sheet or full-screen fallback over a cramped centered modal
- destructive confirms should lead with consequence clarity, not decorative UI
- if the overlay edits existing data, keep the relationship to the source record obvious
