# Detail Surface Contract

Define the detail surface contract before composing sections.

Clarify:

- which entity, record, or object the page represents
- who arrives on the page and why
- what must be understood in the first viewport
- what primary action or primary read should happen first
- which metadata is essential versus secondary
- which related sections matter: history, members, notes, logs, attachments, purchases, or connected entities
- which states matter: loading, not found, empty related data, error, gated access, or unavailable actions

Prefer the smallest detail page that completes the user's main job clearly. Add related sections only if they reinforce the main record and do not compete with it.
