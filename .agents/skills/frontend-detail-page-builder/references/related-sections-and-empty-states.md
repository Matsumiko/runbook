# Related Sections And Empty States

Treat related sections as supporting evidence for the main record.

Common supporting sections include:

- activity or audit timeline
- related items or linked entities
- notes or comments
- attachments
- members or owners
- purchases, invoices, or transactions
- logs, events, or operational history

Use rules like these:

- keep the page summary dominant; related sections should not overshadow it
- use tabs only when the page would otherwise become too dense
- use stacked sections when the default reading order matters more than quick switching
- make empty related sections informative instead of blank
- if the record is missing or inaccessible, show that state before rendering empty shells for unrelated sections

Non-happy-path checklist:

- loading summary state is visible and stable
- not-found or deleted-record treatment is explicit
- permission-limited actions are explained, not silently missing without context
- empty related data still shows what the section is for and what to do next
- narrow layouts keep the main record identity visible before secondary history or side content
