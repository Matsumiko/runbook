# Selection, Preview, and Detail Pane

Master-detail UX depends on retained context.

Guidelines:

- selected state must be easy to spot but not overpower the list
- preview text should help scanning without duplicating the detail pane
- detail panes should answer the next decision quickly
- keep destructive or secondary actions subordinate to the selected record summary
- avoid making the detail pane so rich that the master list becomes irrelevant

If the detail pane cannot support the task honestly, prefer a dedicated route.
