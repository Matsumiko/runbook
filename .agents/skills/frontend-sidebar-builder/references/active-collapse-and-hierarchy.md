# Active, Collapse, and Hierarchy Rules

Treat active state and collapse behavior as core product behavior.

Check:

- active item styling remains obvious without becoming louder than the page title
- parent groups explain which child route is active
- collapsed rails still preserve meaning through icons, labels, or tooltips
- unread counts, badges, and status pills do not overpower nav labels
- keyboard focus remains visible in expanded and collapsed states

Prefer:

- one dominant active marker
- stable grouping order
- explicit collapse rules instead of ad-hoc hiding
