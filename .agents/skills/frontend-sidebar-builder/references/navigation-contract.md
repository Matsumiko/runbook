# Sidebar Navigation Contract

Define the sidebar contract before writing UI.

Clarify:

- what the sidebar navigates
- whether it is global app navigation or local section navigation
- which destinations are primary, secondary, and utility-only
- whether route depth affects active highlighting
- whether workspace, project, or org switching belongs in the shell
- whether permissions or plans hide, disable, or label items

Good contracts separate:

- primary destinations
- contextual section links
- workspace switching
- settings or account utilities
- destructive or sign-out actions
