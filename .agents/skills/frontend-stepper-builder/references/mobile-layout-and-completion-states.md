# Mobile Layout and Completion States

Desktop steppers often collapse badly on mobile.

Decide:

- whether mobile keeps labels, shows numbers only, or collapses to a compact progress summary
- how users understand current stage when horizontal space disappears
- how the final completed state differs from an in-progress last step
- whether completion triggers a new summary, confirmation, or next-action callout
- how optional or skipped steps remain understandable in condensed layouts

Prefer a compact but honest mobile pattern over squeezed desktop markup.
