# Marker Cluster And Selection Model

Map density should stay understandable under real result counts.

## Marker clarity

- Use marker styles that fit the product's semantic language.
- Keep selected marker state visibly distinct.
- Avoid too many simultaneous marker styles.

## Clustering

- Cluster when marker density overwhelms the map.
- Keep cluster meaning simple and discoverable.
- Preserve the user's sense of where selected detail came from.

## Selection model

- Synchronize selection between marker, list row, and detail panel when those surfaces coexist.
- Do not let selection disappear ambiguously after pan or zoom changes.
