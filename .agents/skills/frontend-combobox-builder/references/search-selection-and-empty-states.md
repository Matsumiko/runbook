# Search Selection And Empty States

Typed selection must keep user intent visible.

## Query versus selection

- The user should be able to distinguish current typing from committed value.
- Multi-select fields should make selected tokens feel stable while new input continues.
- When grouping or ranking is involved, show enough metadata to explain why an option appears.

## Empty states

- Show a clear no-results state when filtering eliminates all options.
- Distinguish "no results" from "start typing" and from "loading."
- If creation is allowed, empty states should present that path clearly rather than ambiguously.

## Large datasets

- Do not overwhelm users with long flat lists.
- Use grouping, secondary text, avatars, badges, or recent items only when they improve decision-making.
