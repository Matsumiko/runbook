# Timeline Contract

Use this reference to define what kind of chronology the timeline represents.

## Common timeline jobs

- Activity feed or audit trail
- Order, shipment, or incident progression
- Roadmap or milestone sequence
- Release or changelog chronology

## Contract questions

Clarify:

- newest-first, oldest-first, or milestone order
- event grouping by date, actor, or status
- major versus minor entries
- expandable details or compact entries
- empty-history behavior

## Pattern boundary

- Use calendar-builder for planning and scheduling grids.
- Use detail-page-builder when the timeline is only one section inside a broader record route.
