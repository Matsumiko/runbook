# Density Empty States And Responsive Flow

Timelines can become noisy or empty with little warning.

## Dense history

- Compress repeated low-signal events when the list gets long.
- Preserve key timestamps and major milestones even when compacting.

## Empty states

- Distinguish no history yet from unavailable history.
- Give the user context about what should appear here next.

## Responsive flow

- Horizontal timelines often need to collapse to vertical flow on smaller screens.
- Keep chronology readable after collapse without relying on decorative connectors alone.
