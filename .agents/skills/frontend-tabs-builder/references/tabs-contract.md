# Tabs Contract

Define the tab system before building UI:

- what each tab represents
- whether switching changes route state, local panel state, or filtered content
- default tab and deep-link behavior
- whether panel state persists or resets on switch
- whether badges, counts, or status indicators belong in the label
- expected number of tabs and overflow risk
- responsive fallback when the full tab bar does not fit

Do not choose the tab chrome first and retrofit the interaction model afterward.
