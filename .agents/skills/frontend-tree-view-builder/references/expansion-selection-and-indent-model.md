# Expansion, Selection, and Indent Model

Tree readability depends on disciplined row structure.

Guidelines:

- keep caret placement and indentation consistent at every level
- make expandable rows visibly different from leaves
- separate selected state from expanded state
- keep badges, counts, and action icons subordinate to the label
- avoid arbitrary indentation jumps that break hierarchy scanning
- use connector lines only if they materially help in the current product language

If the tree includes checkboxes or secondary actions, keep the tap or click targets clearly separated.
