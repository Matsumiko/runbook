# Tree Contract

Define the tree surface before choosing UI chrome.

Capture:

- node type and label model
- depth rules and root state
- child presence model: known children, lazy children, or leaf nodes
- selection mode: none, single, multi, or checkbox selection
- expansion state ownership: local state, URL state, persisted state, or controlled prop
- row actions, badges, counts, or drag affordances
- state coverage: loading node, empty branch, error branch, filtered tree, selected node

Make it explicit whether the tree is for navigation, editing, browsing, or choosing.
